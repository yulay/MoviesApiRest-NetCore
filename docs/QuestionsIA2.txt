Pregunta 2 - MongoDB sin instalación local
------------------------------------------
RECOMENDACION: Usar MongoDB Atlas (versión cloud gratuita)

MongoDB Atlas ofrece un tier gratuito (M0 Sandbox) que incluye:
- 512 MB de almacenamiento (suficiente para desarrollo y pruebas)
- Sin costo
- No requiere instalación local
- Accesible desde cualquier lugar
- Backups automáticos

PASOS A SEGUIR:
1. Ir a https://www.mongodb.com/cloud/atlas/register
2. Crear una cuenta gratuita (puedes usar Google o email)
3. Crear un nuevo proyecto (ej: "MoviesApiRest")
4. Crear un cluster gratuito (M0 Sandbox):
   - Seleccionar "FREE" tier
   - Elegir el proveedor cloud más cercano (AWS, Google Cloud o Azure)
   - Elegir la región más cercana a tu ubicación
5. Configurar acceso:
   - En "Database Access": crear un usuario con password
   - En "Network Access": agregar tu IP (o 0.0.0.0/0 para permitir todas durante desarrollo)
6. Obtener el connection string:
   - Click en "Connect" > "Connect your application"
   - Copiar el connection string (formato: mongodb+srv://usuario:password@cluster.xxxxx.mongodb.net/MoviesDb)
7. Compartirme el connection string para configurarlo en el proyecto

Pregunta 7 - Cache: IMemoryCache vs Redis
-----------------------------------------
RECOMENDACION: Usar IMemoryCache

Razones:
1. SIMPLICIDAD: No requiere infraestructura adicional, viene integrado en .NET
2. COSTO: Gratuito, sin servicios externos
3. RENDIMIENTO: Para una sola instancia de la aplicación es muy rápido
4. ESCALA DEL PROYECTO: Para un catálogo de películas con uso moderado es más que suficiente
5. DESARROLLO: Más fácil de configurar y debuggear

¿Cuándo usaría Redis en su lugar?
- Si tuvieras múltiples instancias de la API (load balancing)
- Si necesitaras cache persistente entre reinicios
- Si el volumen de datos cacheados fuera muy grande

Para este proyecto, IMemoryCache es la opción correcta.
